📖 Complete Setup Guide

Full walkthrough for users who have received access. From launch to autonomous FS in under 5 minutes.

1

Launch the EXE

Run FlyffSmartFS.exe. Windows Defender may show a warning — click "More info" → "Run anyway" (see Defender guide below).

The GUI shows your license state at the top.

2

Open Browsers

In the GUI, click "Open FS Browser" and "Open Main Browser". This launches Chrome/Edge in debug mode and opens Flyff Universe directly.

Already have Flyff open? The bot detects already-running Flyff windows — including installed apps (PWAs). On first scan or start, it offers a one-click restart with debug port. Your login stays intact.

3

Log into Flyff

In both browser windows: navigate to Flyff Universe, log in, press F11 for Fullscreen (1920x1080 recommended).

Make sure FS and Main are in the same party. On your FS browser:

  • Open the party window with HP bars on the LEFT side
  • Position names centered and readable
  • Open the character info window (press T)
  • Hide HP numbers in char info (only show the bar)
4

Enter Character Names

In the GUI "Setup" tab, type your FS Name and Main Name exactly as they appear in-game.

⚠️ Case-sensitive! "MyChar" ≠ "mychar" ≠ "MYCHAR". No trailing spaces.

5

Scan Names (10 seconds)

Make sure your FS is at 100% HP, then click "SCAN NAMES":

  • Phase 1: Click on Main's name in the party list → green rectangle
  • Phase 2: Click on FS name → blue rectangle
  • Phase 3: View switches to char info → click your character name → orange rectangle

A verification popup confirms HP/MP read. Re-scan any time if the UI layout changes.

6

Change Deselect Target Key

In Flyff's game settings (ESC → Settings → Controls): change "Deselect Target" from ESC to ^ (key left of 1).

⚠️ ESC is bugged in Flyff Universe — it opens menus and interferes with the bot. The ^ key works cleanly.

7

Configure & Start

In the "Skills Main" tab:

  • Enable Heal or Heal Rain (never both!)
  • Enable the buffs you want and enter matching Flyff hotkeys
  • Enable Auto Rebuff and Auto Follow

Hit START. Your FS is now autonomous.

Killswitch (ALT) toggles on/off — same key starts and stops. Configurable in the Setup tab.

⚠️ Windows Defender Guide

SmartFS is not code-signed — Defender may flag it as suspicious. This is a false positive (typical for unsigned Python EXEs).

When launching the EXE:

  1. Windows shows "Windows protected your PC"
  2. Click "More info"
  3. Click "Run anyway"

Add to Defender exclusions (recommended):

  1. Open Windows Security (search in Start menu)
  2. Go to Virus & threat protection
  3. Click Manage settings under "Virus & threat protection settings"
  4. Scroll to Exclusions → click Add or remove exclusions
  5. Click Add an exclusion → Folder
  6. Select the folder where FlyffSmartFS.exe is located
  7. Done — Defender will no longer interfere.

If Defender already deleted the EXE:

  1. Open Windows Security
  2. Go to Protection history
  3. Find the SmartFS entry → click Actions → Allow on device
  4. Request the EXE again

What resolution is supported?

1920x1080 Fullscreen is the main target. 1440p and 4K are also supported — the scan dialog and bar measurements scale proportionally.

Can I use my PC while it runs?

Yes, but Flyff must stay in the foreground. ALT+TAB away or minimizing the Flyff browser triggers Chromium's WebGL background throttling — the bot sees stale pixels. Use a second monitor or position the bot GUI next to Flyff.

Do I need both accounts on one PC?

Yes. Both Flyff browsers run on the same machine. Multi-PC setups are not supported.

Chrome or Edge?

Both work, including Flyff as an installed app (PWA). The bot auto-detects already-running instances and offers a one-click relaunch with debug port.

Can I use Flyff as an installed app?

Yes. Limitation: if BOTH chars are PWAs, Chromium shares one process — use mixed mode (one PWA + one regular tab) or two different browsers.

Is it detectable?

SmartFS uses CDP (Chrome DevTools Protocol) for input — events land directly in the browser's JS engine, no OS-level keystrokes. No memory injection, no DLL hooking. Botting still violates Flyff ToS — use at your own risk.

What about Holy Cross?

HC is in Alpha. It uses a custom YOLO model (~200 MB one-time download). Works best with FS close to Main, camera zoomed out, monsters that live 5+ seconds.

Do I get support?

Does it work on private servers?

SmartFS is built for Flyff Universe (browser version). Private servers with different clients are not supported.
